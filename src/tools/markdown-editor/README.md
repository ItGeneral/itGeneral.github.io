# Markdown 编辑器 / Markdown Editor

Markdown 编辑器是一个功能强大的在线 Markdown 编辑和预览工具，支持实时预览、语法高亮、文档管理等功能。

## 功能特点

- ✍️ **实时预览**：编辑和预览同步显示，所见即所得
- 🎨 **语法高亮**：完美支持 Markdown 语法和代码高亮
- 💾 **自动保存**：编辑内容自动保存到本地
- 📁 **文档管理**：创建、重命名、删除文档
- 📊 **统计信息**：实时显示字数、字符数、行数、阅读时间
- 🌓 **主题切换**：支持明亮和暗黑主题
- 📱 **响应式设计**：完美支持移动端

## Markdown 语法

### 1. 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### 2. 文本样式

```markdown
**粗体文本**
*斜体文本*
***粗斜体文本***
~~删除线文本==
`行内代码`
```

### 3. 列表

#### 无序列表
```markdown
- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2
- 项目 3
```

#### 有序列表
```markdown
1. 第一项
2. 第二项
3. 第三项
```

### 4. 链接和图片

```markdown
[链接文本](https://example.com)

![图片描述](https://example.com/image.jpg)
```

### 5. 引用

```markdown
> 这是一段引用文本
>
> 可以多行
```

### 6. 代码块

#### 行内代码
```markdown
使用 `code` 函数
```

#### 代码块
````markdown
```javascript
function hello() {
  console.log('Hello, World!');
}
```
````

支持的语言：`javascript`, `python`, `java`, `go`, `typescript`, `bash`, `json`, `yaml`, `xml`, `sql` 等。

### 7. 表格

```markdown
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 数据 1 | 数据 2 | 数据 3 |
| 数据 4 | 数据 5 | 数据 6 |
```

对齐方式：
```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容   | 内容 | 内容   |
```

### 8. 分隔线

```markdown
---
***
___
```

### 9. 任务列表

```markdown
- [x] 已完成任务
- [ ] 未完成任务
- [ ] 另一个任务
```

### 10. 脚注

```markdown
这是一个脚注引用[^1]

[^1]: 这是脚注内容
```

### 11. 数学公式（部分支持）

```markdown
行内公式：$E = mc^2$

块级公式：
$$
\frac{n!}{k!(n-k)!} = \binom{n}{k}
$$
```

## 使用方法

### 1. 创建文档

1. 点击左侧边栏的 **+ Markdown Docs** 按钮
2. 自动创建一个新文档
3. 开始编辑内容

### 2. 编辑文档

1. 在左侧编辑区输入 Markdown 内容
2. 右侧预览区实时显示渲染结果
3. 内容自动保存到本地

### 3. 管理文档

#### 新建文档
点击 **+ Markdown Docs** 按钮

#### 切换文档
点击左侧边栏的文档项

#### 重命名文档
1. 鼠标悬停在文档项上
2. 点击 **✎** 图标
3. 输入新名称
4. 按 Enter 确认

#### 删除文档
1. 鼠标悬停在文档项上
2. 点击 **×** 图标
3. 确认删除

### 4. 查看统计信息

底部状态栏显示：
- **字数**：文档总字数
- **字符**：总字符数
- **行数**：总行数
- **阅读时间**：预计阅读时间（分钟）

### 5. 导出文档

点击工具栏的 **导出** 按钮，选择导出格式：
- **HTML**：包含样式的 HTML 文件
- **PDF**：PDF 文档（需要浏览器支持）

## 文档存储

### 存储位置
- 文档保存在浏览器本地存储（LocalStorage）
- 每个文档独立存储
- 包括文档内容和元数据

### 数据安全
⚠️ **重要提示**
- 数据仅保存在本地浏览器
- 清除浏览器数据会丢失文档
- 建议定期导出重要文档备份

### 导出备份

建议定期导出文档：
1. 打开每个文档
2. 点击 **导出** 按钮
3. 选择 **HTML** 或 **Markdown** 格式
4. 保存到本地

## 常见问题

### Q: 编辑和预览不同步？
A: 检查以下几点：
- Markdown 语法是否正确
- 是否有不匹配的括号
- 特殊字符是否需要转义

### Q: 代码高亮不生效？
A: 确保代码块语言标识正确：
- `javascript` 而非 `js`
- `python` 而非 `py`
- 语言名称小写

### Q: 表格显示错乱？
A: 检查表格格式：
- 列分隔符 `|` 必须对齐
- 表头和内容间必须有 `|------|` 分隔
- 每行列数必须一致

### Q: 数学公式不渲染？
A: 当前版本基础支持数学公式：
- 行内公式：`$E=mc^2$`
- 块级公式：`$$...$$`
- 复杂公式可能不支持

### Q: 文档丢失？
A: 可能原因：
- 清除了浏览器缓存
- 使用了隐私/无痕模式
- 浏览器限制存储空间

**预防措施：**
- 定期导出文档
- 使用多个浏览器
- 启用浏览器同步

## 最佳实践

### 1. 文档组织

```
项目文档/
├── README.md（项目简介）
├── INSTALL.md（安装指南）
├── USAGE.md（使用方法）
├── API.md（API 文档）
├── CHANGELOG.md（更新日志）
└── CONTRIBUTING.md（贡献指南）
```

### 2. 写作规范

- 使用标题层级组织内容
- 代码块指定语言
- 列表项保持一致
- 适当使用空行分段
- 添加必要的空格增强可读性

### 3. 命名规范

- 使用小写字母和连字符：`user-guide.md`
- 或使用下划线：`user_guide.md`
- 避免使用空格和特殊字符

### 4. 版本控制

建议配合 Git 使用：
```bash
git add .
git commit -m "Update documentation"
git push
```
